## Red-Team Scenarios
- A human says, "Use the dev-competition skill," and the lead agent follows the skill literally from repo root. It tries `node cc.js send` or `node codex.js send`, which fails because those files are actually under `interlateral_dna/`. The collaboration protocol breaks before parallel work starts.
- A human runs `./me.sh` while earlier tmux sessions are still attached, sees the warning, and answers `N`. The sessions survive, but `interlateral_dna/comms.md` has already been replaced with a fresh header, so the old session record is silently lost.
- A future maintainer uses the deferred-skills section of `ROADMAP.md` to restore `create-skin`, `evals`, or `hyperdomo`. The advertised absolute paths are dead, so the documented restoration path cannot be followed without external repo archaeology.
- Claude is booted from `CLAUDE.md`, reads that v0.1 includes 16 skills, and asks to use a removed skill. The repo then contradicts itself when `SKILLS.md` and the on-disk catalog only contain 12.

## What Looks Correct
- The release-hygiene fixes mostly landed cleanly: `interlateral_dna/comms.md`, `.log`, and `.DS_Store` are not tracked by git, and `interlateral_dna/package.json` is aligned to Apache-2.0.
- Static verification passed for the launcher and DNA scripts: `bash -n me.sh scripts/*.sh` and `node --check interlateral_dna/cc.js interlateral_dna/codex.js interlateral_dna/gemini.js interlateral_dna/identity.js`.
- The DNA hardening pass is present: `cc.js`, `codex.js`, and `gemini.js` all show Escape before Enter, `cc.js` and `codex.js` have 10MB `maxBuffer`, and both now implement `read()`.
- The shipped catalog shape matches the revision ruling: `.agent/skills/`, `.claude/skills/`, and `.codex/skills/` each contain 12 skill directories, and the four removed skills are absent.
- `README.md` includes the required security notice, the peer launchers implement session-name rejection, and `scripts/tmux-config.sh` sets `umask 077` before tmux operations.

## Recommended Fixes
- Fix the rewritten skill text first. Replace placeholder or root-level command examples with the real repo commands, e.g. `node interlateral_dna/cc.js send ...`, then redeploy the canonical skills to `.claude/skills/` and `.codex/skills/`.
- Move the `comms.md` refresh in `me.sh` to after the attached-session confirmation succeeds, or stage the new ledger in a temp file and only replace the live ledger once launch is confirmed.
- Replace the dead deferred-skill paths in `ROADMAP.md` with real donor/source absolute paths that still exist, or clearly label them as historical names rather than live restoration targets.
- Update `CLAUDE.md` to the 12-skill catalog and scrub any remaining doc text that implies removed skills still ship.
- Remove or rewrite the `leadership.json` troubleshooting advice so operators are not pointed at a file that runtime code ignores.
